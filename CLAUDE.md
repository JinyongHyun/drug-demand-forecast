# 약물 수요 예측 데이터 분석 프로젝트

## 프로젝트 목적
지역별(시군구) 약물 처방 수량 및 금액 데이터와 인구 데이터를 결합하여,
지역별 약물 수요를 탐색하고 시계열 예측 모델을 구축한다.

- 주요 분석 약물: A10A (인슐린 및 유사체), A10B (혈당강하제, 인슐린 제외)
- 예측 목표: 시군구 단위 월별 처방 수량 / 금액

## 분석 단계

### 1단계: EDA (탐색적 데이터 분석)
- 각 파일 로드 및 구조 파악 (결측치, 이상치, 데이터 타입)
- 시군구별 월별 처방 수량 추세 시각화
- A10A vs A10B 비교 분석
- 인구 규모 대비 처방량 분포 확인
- 상병별 처방 순위 분석

### 2단계: 전처리
- 멀티헤더 엑셀 → tidy 형태 변환 (wide → long)
- 기간 컬럼 datetime 변환
- 인구 데이터와 처방 데이터 시군구 기준 병합
- 1인당 처방량 등 파생 변수 생성
- 결측치 처리 및 이상치 제거

### 3단계: 모델링
- 특성 엔지니어링 (lag features, rolling mean, 인구 특성 등)
- 시계열 예측 모델 학습 (기본: Linear Regression → 고급: XGBoost / LightGBM)
- 시군구별 개별 모델 또는 패널 데이터 모델 검토

### 4단계: 평가
- 평가 지표: MAE, RMSE, MAPE
- 시각화: 예측 vs 실제 비교, 잔차 분석
- 지역별 예측 성능 차이 분석

## 코딩 규칙

1. **파일 구조**: 단계별로 스크립트 분리 (`01_eda.py`, `02_preprocessing.py`, `03_modeling.py`, `04_evaluation.py`)
2. **출력 저장**: 시각화 결과는 `output/figures/`, 처리된 데이터는 `output/data/`에 저장
3. **한글 주석**: 코드 설명은 한국어로 작성
4. **인코딩**: 파일 저장 시 `encoding='utf-8-sig'` 사용 (Excel 호환)
5. **재현성**: 난수 시드 고정 (`random_state=42`)
6. **변수명**: snake_case 사용, 의미 있는 한글/영문 혼용 허용
7. **plt.show() 금지**: 시각화는 반드시 파일로 저장 후 경로 출력
8. **엑셀 헤더**: 원본 파일은 3~4행 멀티헤더 구조 → `header=None`으로 읽고 수동 파싱
